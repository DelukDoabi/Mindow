## 2. What went well

1. **Single-boundary + nullable-client pattern** (`AuthRepository`, `OnboardingRepository`) paid off across the whole epic. Graceful no-backend degradation, Supabase-free screen tests via provider override тАФ reused unchanged from Story 1.4 through 1.7.
2. **Hive typeId registry + CI gate** (1.1): protects offline data integrity on every PR with zero further touches; ready for the Epic 2 event sourcing.
3. **Aurore design system as single source of truth**: no literals in UI, en+fr i18n from day one, visual consistency across 5 screens.
4. **Screen-level testing discipline** (fake repository): fast, stable, Supabase-free тАФ grew 0 тЖТ 33 tests with no flakiness.
5. **Flash-free synchronous routing** (1.5): seeding `onboardingCompleteProvider` in `bootstrap.dart`, GoRouter never rebuilt тЖТ navigation stack preserved across sign-in/out.

**boss's headline (what to reproduce):** *the fluidity* тАФ the linear, momentum-preserving cadence of create-story тЖТ dev тЖТ validate тЖТ finalize тЖТ commit/push тЖТ next, with each story building cleanly on the last.

## 3. What hurt (recurring frictions)

1. **`git commit` hang under PowerShell/PSReadLine** with multi-line messages containing `()` / apostrophes тЖТ worked around with a temp message file (`git commit -F`). Recurred on **every** commit. *[boss flagged as priority]*
2. **Stale terminal scrollback**: `get_terminal_output` returned the old buffer during validation тЖТ false reads, extra round-trips.
3. **Flutter PATH not inherited**: every flutter/dart command had to be prefixed with `$env:Path = "C:\src\flutter\bin;" + $env:Path;`. *[boss flagged as priority]*
4. **Strict lints discovered at runtime**: `avoid_escaping_inner_quotes` (FR apostrophes тЖТ double quotes), `cascade_invocations`, `use_build_context_synchronously`.
5. **Deliberately deferred debt тЖТ Epic 2**: `hive_generator` (analyzer incompatibility) тЖТ adopt `hive_ce_generator` fork; Inter font via `google_fonts` instead of bundled TTFs; iOS schemes not validated (no macOS host); GDPR Edge Functions *scaffolded, not deployed* (no Supabase project linked); server-side cross-device restore.

---

## 4. Key lessons

1. The **nullable-client / fake-repository** pattern is the project's backbone тАФ keep every Supabase touch behind a repository that tolerates a `null` client, so screens build and test without a backend.
2. **Tooling friction compounds.** Per-command PATH prefixing and the commit hang were small each time but recurred 7├Ч. Worth fixing durably before Epic 2.
3. **Lint rules should be front-loaded**, not discovered per story. A repo-memory lint memo prevents re-learning the same rules.
4. **Scaffold-then-defer is a valid GDPR/infra strategy** тАФ Epic 1 laid contracts (Edge Functions, registry, event contract stub) that Epic 2 completes with real data.
