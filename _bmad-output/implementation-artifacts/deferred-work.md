# Deferred Work

## Deferred from: code review of 2-1-event-sourced-sync-engine-foundation (2026-06-07)

- **Outbox box never opened in app / no EventStore provider** ‚Äî AC#1's runtime wiring: nothing opens `Box<OutboxRecord>` at app startup and no Riverpod provider exposes `EventStore`/`SyncQueue` (engine is test-injectable only). Deferred to Story 2.2: no event producer exists before 2.2, so the box-open (in `bootstrap.dart` after `registerAdapters()`) + provider wiring lands with the first emitter (`preoccupation.captured`).
- **`SyncQueue.flush()` partial-failure / retry semantics** ‚Äî `flush()` marks records `sent` then awaits `client.push()`; if push throws, records stay `sent` (re-picked up by the next `pending()`/flush, which is offline-tolerant) but the exception propagates with no backoff/retry policy. Revisit when the real reconciliation transport is wired in Story 2.2.
- **Harden envelope deserialization at the backend boundary** ‚Äî `EventEnvelope.fromJson` and `OutboxRecord.toEnvelope` assume well-formed JSON (missing keys, non-Map payload, non-numeric `schema_version`, malformed ISO-8601, malformed `payloadJson`). Current inputs are engine-produced or test fixtures; add contextual `FormatException` validation at the real (untrusted) backend boundary introduced in Story 2.2.
- **Contract-parity regex robustness** ‚Äî `event_contract_parity_test.dart` extracts keys via a `dotAll` regex over the `EVENT_ENVELOPE_KEYS` block; a quoted string inside a comment within the array literal could be captured. Minor test-only hardening (strip comments before extracting keys).

## Deferred from: code review of stories 2-2, 2-3, 2-4 (2026-06-10)

- **[2-3] Category tokens hardcoded in two places** ‚Äî The fixed nine category tokens (`'Administratif'`, `'Famille'`, etc.) are defined independently in `supabase/functions/ai-analyze/index.ts` (CATEGORIES array) and `lib/features/brain_dump/presentation/home_screen.dart` (_categoryLabel switch). A category rename requires edits in both; no compile-time guarantee of sync. Same repo, low risk for MVP ‚Äî extract to a shared canonical list when a category change is first needed.
- **[2-3] `AnalysisService` hardcoded `languageCode = 'fr'`** ‚Äî The `analysisService` provider constructs `AnalysisService` with the default French locale; English-device users get French-calibrated crisis/weighing prompts. Intentional French-first MVP default. Wire device locale when English-first markets are targeted.
- **[2-4] `ReplayEngine._compare` UUID tiebreaker on same-millisecond local events** ‚Äî When two local events share the same `createdAt` millisecond, ordering falls to random UUID v4 `eventId` lexicographic comparison, potentially producing a non-causal replay order. The `createdAt` fix from Story 2.4 already reduces risk significantly. Resolve if high-frequency event emission (e.g., bulk capture) is introduced.
